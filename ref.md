# C 문법 오류 개념 레퍼런스

---

## 함수 반환 {#함수-반환}

- `void` 함수는 값을 반환하면 안 됩니다.
- `int` 함수는 반드시 `return 값;` 형태로 값을 반환해야 합니다.

### ✅ 올바른 예시

```
int foo() {
    return 1;
}
```

### ❌ 잘못된 예시

```
void foo() {
    return 1; // 오류: void 함수는 값을 반환할 수 없습니다
}
```

---

## 변수 선언 {#변수-선언}

- 변수를 사용하기 전에 반드시 선언해야 합니다.
- 선언하지 않은 변수를 사용할 경우 컴파일 오류가 발생합니다.

### ❌ 잘못된 예시

```
int main() {
    x = 10; // 오류: x가 선언되지 않음
    return 0;
}
```

### ✅ 올바른 예시

```
int main() {
    int x = 10;
    return 0;
}
```

---

## 입출력 형식 지정자 {#입출력-형식}

- `scanf`에서 입력받을 때 변수의 주소를 넘겨야 하며 형식 지정자와 타입이 일치해야 합니다.

### ❌ 잘못된 예시

```
int x;
scanf("%d", x); // 오류: 주소연산자(&) 빠짐
```

### ✅ 올바른 예시

```
int x;
scanf("%d", &x);
```

---

## 포인터 {#포인터}

- 포인터는 선언과 동시에 초기화되지 않으면 쓰기 전에 주의가 필요합니다.
- 포인터 타입이 맞지 않으면 오류 발생 가능성이 큽니다.

### ❌ 잘못된 예시

```
int *p;
*p = 10; // 오류: 초기화되지 않은 포인터 역참조
```

### ✅ 올바른 예시

```
int x = 10;
int *p = &x;
*p = 20;
```

---

## 배열 인덱스 초과 {#배열}

- 배열의 인덱스는 0부터 시작하며, 선언한 크기 이상 접근 시 오류 발생

### ❌ 잘못된 예시

```
int arr[3];
arr[3] = 5; // 오류: 인덱스 3은 배열 크기 초과
```

### ✅ 올바른 예시

```
int arr[3];
arr[2] = 5; // 최대 인덱스는 2
```

---

## 세미콜론 누락 {#기본문법}

- C에서 문장 끝에는 반드시 `;`가 있어야 합니다.

### ❌ 잘못된 예시

```
int x = 10 // 오류: 세미콜론 빠짐
int y = 20;
```

### ✅ 올바른 예시

```
int x = 10;
int y = 20;
```

---

## 비교 연산자 {#연산자}

- `if (a = b)` 는 값을 대입하는 것이며, 비교하려면 `==` 사용해야 합니다.

### ❌ 잘못된 예시

```
if (a = b) // 오류: 대입 연산자 사용
```

### ✅ 올바른 예시

```
if (a == b) // 비교 연산자 사용
```

---

## 무한 루프 {#반복문}

- 반복문에서 종료 조건이 없는 경우 무한 루프가 발생할 수 있습니다.
- 의도적이지 않은 무한 루프는 논리적 오류입니다.

### ❌ 잘못된 예시

```
while (1) {
    // 조건 없이 계속 반복됨
}
```

### ✅ 올바른 예시

```
int i = 0;
while (i < 10) {
    i++;
}
```

---

## 괄호 닫힘 오류 {#괄호}

- 함수나 조건문, 반복문에서 괄호가 열리고 닫히지 않으면 문법 오류 발생

### ❌ 잘못된 예시

```
if (x > 0 {
    printf("양수입니다");
```

### ✅ 올바른 예시

```
if (x > 0) {
    printf("양수입니다");
}
```

### ❌ 잘못된 예시

```
struct Student;
Student s1; // 오류: Student 타입 정의가 없음
```

### ✅ 올바른 예시

```
struct Student {
    int id;
    char name[20];

struct Student s1;
s1.id = 123;
```

## 메모리 동적 할당 오류 {#malloc}

- `malloc` 사용 시 반환된 포인터의 타입에 맞게 캐스팅해야 하며, NULL 체크가 필요합니다.

### ❌ 잘못된 예시

```
int *p;
p = malloc(sizeof(int) * 10); // 오류: stdlib.h 누락, 캐스팅 없음
```

### ✅ 올바른 예시

```
#include <stdlib.h>

int *p = (int *)malloc(sizeof(int) * 10);
if (p == NULL) {
    // 메모리 할당 실패 처리
}
```
